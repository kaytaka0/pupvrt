### 環境変数の読み込み処理

Gemini API 実行時に API キーを環境変数から読み込む想定だったが、なぜか環境変数の読み込み処理の部分が削除されていた。なんでや

環境変数は dotenv で読み込むようにした。


dotenv のインストール時に気がついたのだが、npm install がエラーで動かない。
npm パッケージの peerdependency 関連でエラーが出てる様子

調べてみると
- このプロジェクトで指定している pupetter のバージョン
- puppeteer-screen-recorder が要求しているバージョン
が違うことが原因だった。


回避策としては２種類あったが、package.json の overrides で puppeteer のバージョンを指定することで回避した。


実際には回避策が二種類あった。
- npm コマンドでlegacy-peer-deps オプションを付ける方法。。。つまりライブラリが使用するパッケージのバージョンを無視する
- package.json の overrides で puppeteer のバージョンを指定する方法。。。こちらはプロジェクトで使用する puppeteer のバージョンを固定することを宣言する

overrides で宣言することで、このプロジェクトでは新しい puppeteer を使うことにする。
そのため、puppeteer-screen-recorder がこれからも正常に動くかどうかは運次第。。。
